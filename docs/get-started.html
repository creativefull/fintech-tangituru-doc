Sebelum Memulai

Sebelum memulai menggunakan project ini pastikan anda sudah membaca peraturan cara akses API di project ini

Client ID

Client ID adalah berupa huruf2 random unik yang di generate dari system kami. anda bisa membuatnya di halaman dashboard anda

END POINT

Aplikasi ini mempunya environment end point atau biasa di sebut host, yaitu

http://fintech.tangituru.com/api/v1

Signature

Sebagai Keamanan Kami menggunakan Signature ini untuk kunci yang akan anda kirimkan ke End Point Kami.

Algoritma Signature

-- request_method : GET / POST
-- client_id : YOUR_CLIENT_ID
-- url : URL_ENDPOINT
-- request_body : []
-- request_encrypted : LOWERCASE(hash256(request_body))

// This is for generate signature
hmac256(UPPER(request_method):url:request_encrypted, client_id)

// This is example
hmac256('GET:/api/client/info:4f53cda18c2baa0c0354bb5f9a3ecbe5ed12ab4d8e11ba873c2f11161202b945', 'Njg2NTZjNmM2Zg==')

// Output
ac7b190ebbb8bceaa927713bb4d39575706d9037b6cc3c140dae2b699b0c7982

Penjelasan Algoritma Signature

  • request_metod : metode request yang akan anda gunakan untuk request ke server kami
  • client_id : client id aplikasi anda yang di dapat dari halaman dashboard
  • url : url api yang ingin di akses dari aplikasi anda
  • request_body : data yang akan di kirimkan ke server kami, jika data kosong bisa di masukkan []
  • request_encrypted : request_body harus di encrypt menggunakan algoritma hash256 dan lowerCase

Authentication

Semua URL API kami memerlukan Authentication

Algoritma Authentication

semua data authentication harus dikirim melalui headers

curl -v -H "Accept: application/json" \
        -H "Content-type: application/json" \
        -H "Signature: 0o6Lg.....RSFg=" \
        -H "client_id: Njg2NTZjNmM2Zg==" \
        -X "GET" \
        http://fintech.tangituru.com/api/client/info
